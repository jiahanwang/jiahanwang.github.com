JavaScript

Yes, JavaScript is not hard to get started. But it is never on the list of the possible most easiest programming languages on the earth. Even in real industry, few front-end developers would say they are professional in JavaScript. I started my JS journey since my sophomore year. What really deepened my understanding in JS is the book Professional JavaScript for Wed Developers by Nicholas C. Zakas which I read as preparation for the interview of Baidu Internship. JS is a bizarre language in various kinds of ways. Here are some aspects that I think distinguish JS:

  • Loose typing is not so loose: JS is a loose typing language. However, this doesn’t mean there is no need for us to worry about datatypes. Actually, JS has five basic datatypes: Undefined, Null, Boolean, Number and String; it also has a primitive datatype: Object. Weird things usually happen when transferring between different types especially Undefined, Null and NaN(sub type of Number). We also should focus when we want to use ===and switch() as they do not transfer datatypes automatically.
  • BOM is a BOMB: Another annoying thing about JS is cross-browser programming. A typical JavaScript implementation usually has three parts: Core APIs, Document Object Model (DOM) and Browser Object Model (BOM). The Core APIs follows the ECMAScript standard and hardly varies among browsers. DOM follows W3C standards and behaves similarly (the biggest exception is IE). When it comes to BOM, BANG! Since there is no BOM standard, APIs are implemented according to browser vendors’ tastes. A quick way to solve this problem is to apply a JS framework such as jQuery. We can also build our own utility library since there are many good examples in Professional JavaScript for Web Developers and JavaScript- the Definite Guide.
  • Function is Object: I have to say I regarded JS as a process-oriented language at first.JS supports OOP; it just implements OOP in a different way. Firstly, the key word class doesn’t exist in JS. We should use function to declare a class and then create a object by using the key word new. In addition, the inheritance in JS is prototype-based instead of classic class-based. We often inherit attributes and methods desperately, which is named Combination Inheritance.

It is still a looooong way to be a JS professional. However, I have known what I should do and have started my new journey ;).

CSS2/CSS3

JavaScript is bizarre enough, but CSS is more bizarre. Especially when the web page is required to be compatible with IE6, here comes the nightmare. However thanks to efforts made by non-IE browser vendors,even Microsoft itself, IE6 has gradually stepped out of our view. In fact, I test my web pages primarily in Chrome and Firefox. At last, I would do some extra work for IE8, the default browser of Windows XP. CSS3, the new enhanced CSS, is revolutionary. Here I list some of the new features I favor most:

  • box-pack and box-align: These two feathers are definitely my favorites. CSS is very powerful when compared to the out-of-dated style attribute. But to center an element is surprisingly not an easy work. With these two new feathers, it becomes an easy work.
  • box-shadow and border-radius: These two buddies are the first choice to embellish the web pages. Again, it is much easier than the old image-based shadow and borders.
  • Powerful Selectors: The new CSS3 selectors are very convenient. As supported by jQuery, they are widely used nowadays.

(X)HTML/HTML5

Basically, HTML5 is not the direct descent of XHTML, as XHTML is defined in XML. Additionally, HTML5 doesn’t include the HTML tags alone; it symbols a new system consisting of HTML tags and JavaScript APIs. Here are the new feathers I have applied in my projects:

  • Sematic Tags: It is very confusing to read a web page full of <div>; both for human coders and machine readers. Thus, the sematic tags are very necessary.
  • Video and Audio: Fantastic new functions. We no longer need to write some ugly codes to embed a Flash player into our pages.
  • Cross-document Messaging: Another dream function! No need to message via back-end and JSON can also be used.
  • Web Workers: The definition of thread doesn’t exist in JavaScript, which brings disadvantage to the performance. HTML5 allows us to use Web Workers API to create workers (threads), so that our JS programs could run parallel.
